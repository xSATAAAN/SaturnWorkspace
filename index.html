Local-first control

SATAN Toolkit processes operational records on the user's own device instead of routing workflows through a developer-hosted cloud backend.

Restricted Google scope

When sync is enabled, the app requests only the https://www.googleapis.com/auth/drive.appdata scope. It does not inspect the user's personal Google Drive files.

Structured operational data

Depending on workflow usage, synchronized data may include account records, Gmail entries, IP records, notes, timestamps, workflow states, and per-account browser metadata.

Security Boundary

Device-to-drive synchronization without personal Drive exposure

If the user connects Google Drive, SATAN Toolkit writes only to the app-specific data area within that account. The application does not expose normal Drive files and does not depend on developer-controlled storage infrastructure.

Operational Model

Manual control with optional cloud continuity

Users can connect Google Drive, enable or disable synchronization, push updates manually, and restore local state when needed without exposing general Drive content.
