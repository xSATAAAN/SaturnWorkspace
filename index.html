Desktop App

SATAN Toolkit

SATAN Toolkit is a local desktop application that helps users manage operational account data, Gmail records, IP records, and optional sync backups stored inside the app-specific area of their own Google Drive account.

View Privacy Policy View Terms of Service How It Works

Local-First by Design

SATAN Toolkit runs locally on the user's device. Primary data handling happens on the machine itself, not on an external application server controlled by the developer.

Google Drive Scope

When the user enables cloud sync, the app uses the Google Drive drive.appdata scope only. This app-specific storage is separate from the user's normal Drive files.

No Personal Drive Browsing

SATAN Toolkit does not browse, index, read, modify, or delete the user's general Google Drive files. It only works inside its own hidden application data area.

Data Flow

Direct device-to-drive synchronization

If the user signs in with Google Drive, data moves directly between the local application and the user's own Google Drive app-data folder through Google APIs. The application does not rely on a separate cloud database, third-party analytics service, or developer-owned storage server.

Application Type Local desktop software
Cloud Storage Mode Optional, user-controlled
Google Permission drive.appdata only
Third-Party Analytics None

User Control

Users can choose whether to enable Google Drive sync, manually trigger uploads, sign out at any time, and restore local data from cloud-stored application data when needed.

Scope of Stored Data

Depending on user actions inside the app, synchronized data may include account records, Gmail records, IP records, notes, eligibility states, timestamps, and account-level browser or AdsPower-related metadata required for continuity across devices.
