Local-First Control

SATAN Toolkit runs on the user's own machine. Operational records are processed locally instead of being routed through a developer-hosted cloud backend.

Restricted Google Scope

When the user enables cloud sync, the app requests the Google Drive drive.appdata scope only. This is isolated app storage, not general Drive browsing.

Operational Data Continuity

Depending on user workflows, the synchronized payload may include account records, Gmail entries, IP records, notes, eligibility states, timestamps, and per-account browser metadata.

Compliance Model

Direct device-to-drive synchronization

If the user signs in with Google Drive, the application writes only to the app-specific data area within that Google account. SATAN Toolkit does not expose the user's normal Drive files, and it does not depend on a third-party analytics vendor or a developer-controlled storage server.

User-Controlled Sync

Users can connect Google Drive, toggle sync behavior, push updates manually, sign out, and restore local state from stored cloud payloads when needed.
