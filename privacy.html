Privacy Policy

SATAN Toolkit Privacy Policy

Effective date: April 26, 2026. This Privacy Policy explains how SATAN Toolkit processes local application data and, when enabled by the user, uses Google Drive app-specific storage.

Back to Homepage Terms of Service

1. Overview

SATAN Toolkit is a local desktop application. Its core functionality is performed on the user's own device. The application does not operate a developer-hosted backend for storing the user's operational data.

2. Information Processed by the Application

Depending on how the user uses the application, SATAN Toolkit may process the following categories of data locally:

  • Account records, including email addresses, notes, timestamps, eligibility states, and workflow metadata.
  • Gmail records, including email addresses, usage status, usage timestamps, and optional passwords entered by the user.
  • IP records, including IP addresses, related dates, and associated Gmail references entered or saved by the user.
  • Account-level operational settings such as browser choice, AdsPower profile identifiers, and related proxy configuration metadata where applicable.

3. Google User Data and Google Drive Access

If the user chooses to sign in with Google, SATAN Toolkit requests access only to the Google Drive scope https://www.googleapis.com/auth/drive.appdata.

This means:

  • The application uses only Google Drive's app-specific storage area.
  • The application does not browse, list, inspect, read, modify, or delete the user's normal Google Drive files or folders.
  • The application does not use Google Drive for advertising, profiling, analytics, or unrelated data mining.

4. How Data Is Stored

SATAN Toolkit stores its working data locally on the user's device. If Google Drive synchronization is enabled by the user, the app may also store application data snapshots and related structured records inside the user's own Google Drive app-data area.

Data is transmitted directly between the user's device and Google APIs. SATAN Toolkit does not route this content through a developer-owned database or external storage server.

5. Automatic and Manual Synchronization

Users may enable or disable cloud synchronization from within the application. When sync is enabled, the app may upload updated application data to the user's Google Drive app-data area. Users may also trigger a manual upload.

Signing out of Google Drive inside the application stops future synchronized access until the user signs in again and re-authorizes the connection.

6. No Sale, No Sharing, No Analytics

SATAN Toolkit does not sell user data. It does not share user data with advertisers, data brokers, or analytics networks. The application does not use third-party analytics, tracking pixels, or remote behavioral profiling services.

7. Data Retention and User Control

Local data remains on the user's device until the user edits, removes, migrates, or replaces it. Cloud data stored in Google Drive app-data remains under the user's Google account unless the user deletes it from the application, removes the app's access, or clears the related app-data content from Google Drive.

8. Security

SATAN Toolkit is designed as a local-first application. The app uses Google's OAuth flow for account authorization and stores the resulting Google authorization data locally on the user's machine so the user does not need to sign in again for every session, unless they explicitly sign out or revoke access.

No software can guarantee absolute security. Users remain responsible for securing their device, operating system account, local files, and Google account credentials.

9. Changes to This Policy

This Privacy Policy may be updated if the application's data-handling model changes. The latest published version should be treated as the current policy for SATAN Toolkit.

10. Contact

Questions about this Privacy Policy should be directed through the official SATAN Toolkit distribution or support channel used to provide the application to end users.
