SATAN Toolkit Privacy Policy

Effective date: April 26, 2026. This document describes how SATAN Toolkit processes local application data and, when enabled by the user, stores app-specific payloads in Google Drive.

Back to homepage Open terms of service

1. Overview

SATAN Toolkit is a local desktop application. Its core functionality runs on the user's own device and does not rely on a developer-hosted cloud backend for operational storage.

2. Information Processed by the Application

Depending on how the user operates the application, SATAN Toolkit may process the following categories of data locally:

  • Account records, including email addresses, notes, timestamps, eligibility states, and workflow metadata.
  • Gmail records, including email addresses, usage status, usage timestamps, passwords, recovery emails, and recovery links entered by the user.
  • IP records, including IP addresses, related dates, and associated Gmail references entered or saved by the user.
  • Account-level operational settings such as browser choice, AdsPower profile identifiers, and related proxy configuration metadata where applicable.

3. Google User Data and Google Drive Access

If the user signs in with Google, SATAN Toolkit requests access only to https://www.googleapis.com/auth/drive.appdata.

  • The application uses only Google Drive's app-specific storage area.
  • The application does not browse, list, inspect, read, modify, or delete the user's normal Google Drive files or folders.
  • The application does not use Google Drive for advertising, profiling, analytics, or unrelated data mining.

4. How Data Is Stored

SATAN Toolkit stores its working data locally on the user's device. If Google Drive synchronization is enabled, the app may also store application data snapshots and structured records inside the user's own Google Drive app-data area.

Data moves directly between the user's device and Google APIs. SATAN Toolkit does not route this content through a developer-owned database or remote storage server.

5. Automatic and Manual Synchronization

Users may enable or disable Google Drive sync from within the application. When sync is enabled, the app may upload updated application data to the user's app-data area. Users may also trigger a manual upload.

6. No Sale, No Sharing, No Analytics

SATAN Toolkit does not sell user data. It does not share user data with advertisers, data brokers, or analytics networks. The application does not use third-party analytics, tracking pixels, or behavioral profiling services.

7. Retention and User Control

Local data remains on the user's device until the user edits, removes, migrates, or replaces it. Cloud data stored in Google Drive app-data remains under the user's Google account unless the user deletes it from the application, revokes access, or clears the related app-data content from Google Drive.

8. Security

SATAN Toolkit uses Google's OAuth flow for authorization and stores resulting authorization data locally on the user's machine so repeated sign-ins are not required unless the user explicitly signs out or revokes access.

9. Changes to This Policy

This Privacy Policy may be updated if the application's data-handling model changes. The latest published version should be treated as the current policy for SATAN Toolkit.
